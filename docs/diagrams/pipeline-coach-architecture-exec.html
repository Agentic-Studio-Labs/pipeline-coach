Pipeline Coach Executive View

A daily read-only system that turns CRM hygiene gaps into prioritized AE actions and manager escalations.

Audience: clients and hiring managers
Output: AE brief + manager escalation email
Cadence: daily scheduler or run-once
Business Outcome
AEs know exactly what to do first, every morning.
Risk Reduction
High-value at-risk deals are escalated automatically.
Governance
Read-only CRM behavior with audit records per run.
System Flow
Step 1
Ingest CRM data
Read opportunities, activities, contacts, and owner context from Twenty GraphQL.
Step 2
Detect hygiene issues
Apply configurable YAML rules for staleness, missing fields, and date risk.
Step 3
Recommend next action
Generate concise action suggestions via DSPy (or deterministic fallback templates).
Step 4
Quality-check + retry
Reject weak suggestions, retry per opportunity, then fallback if needed.
Step 5
Prioritize and escalate
Rank by severity and deal value; route critical deals to manager escalation path.
Step 6
Deliver and log
Send briefs via Resend and write per-run/per-issue audit records.
Platform Ownership
  • Workflow: LangGraph (orchestration)
  • Action generation: DSPy + LiteLLM (LLM interface and transport)
  • Business logic: YAML rules + priority scoring (deterministic)
  • Delivery: Resend (email channel)
v1 Boundaries
  • Read-only against CRM (no write-back)
  • No Slack delivery in v1
  • Single-tenant deployment model
  • Designed for extension in v2+
